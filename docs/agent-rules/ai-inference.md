# AI and Inference Rules

Read when the task adds or changes inferential, ranking, classification, summarization, recommendation, or other AI-assisted behavior.

- Keep AI behavior bounded and explicitly approved.
- Prefer factual phrasing over speculation.
- State uncertainty clearly.
- Keep AI output, rule-based logic, source facts, and fallback behavior distinguishable.
- Do not let AI silently override explicit config, source truth, or approved rules.
- New inferential behavior requires Ask-First approval unless already approved by the project.
